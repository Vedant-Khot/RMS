# RMS


# **Task Management API Documentation**

This document provides a guide for interacting with the Task Management API.

**Base URL:** `https://rms-api-karw.onrender.com`

---

## **The Task Object**

The core resource of this API is the Task object.

### Attributes

| Field         | Type     | Description                                                                 | Notes                          |
|---------------|----------|-----------------------------------------------------------------------------|--------------------------------|
| `_id`         | ObjectId | The unique identifier for the task.                                         | Auto-generated by MongoDB.     |
| `description` | String   | The content of the task.                                                    | **Required** when creating.    |
| `completed`   | Boolean  | The status of the task.                                                     | Defaults to `false`.           |
| `assignedTo`  | String   | The person or team the task is assigned to.                                 | Defaults to `'Unassigned'`.    |
| `project_id`  | ObjectId | The ID of the project this task belongs to.                                 | Optional.                      |
| `createdAt`   | Date     | The timestamp when the task was created.                                    | Auto-generated.                |
| `updatedAt`   | Date     | The timestamp when the task was last updated.                               | Auto-generated.                |

---

## **API Endpoints**

### 1. Create a Task

Creates a new task.

-   **HTTP Method:** `POST`
-   **URL:** `/tasks`
-   **Request Body:** A JSON object representing the task.

#### Request Body Example

```json
{
  "description": "Design the new application logo",
  "assignedTo": "Alice",
  "project_id": "636d8b53a89366a315355607"
}
```

#### Success Response

-   **Code:** `201 Created`
-   **Content:** The full task object that was just created.

```json
{
    "_id": "64a2f8b8c5d1e2f3a4b5c6d7",
    "description": "Design the new application logo",
    "completed": false,
    "assignedTo": "Alice",
    "project_id": "636d8b53a89366a315355607",
    "createdAt": "2025-11-07T12:34:56.789Z",
    "updatedAt": "2025-11-07T12:34:56.789Z"
}
```

#### Error Response

-   **Code:** `400 Bad Request` if `description` is not provided.

#### Example cURL

```bash
curl -X POST -H "Content-Type: application/json" \
-d '{"description":"Design the new application logo", "assignedTo":"Alice"}' \
https://your-api-url.onrender.com/tasks
```

### 2. Get All Tasks

Retrieves a list of all tasks.

-   **HTTP Method:** `GET`
-   **URL:** `/tasks`

#### Success Response

-   **Code:** `200 OK`
-   **Content:** An array of task objects.

```json
[
    {
        "_id": "64a2f8b8c5d1e2f3a4b5c6d7",
        "description": "Design the new application logo",
        "completed": false,
        "assignedTo": "Alice",
        "project_id": "636d8b53a89366a315355607",
        "createdAt": "2025-11-07T12:34:56.789Z",
        "updatedAt": "2025-11-07T12:34:56.789Z"
    },
    {
        "_id": "64a2f8c1c5d1e2f3a4b5c6d8",
        "description": "Update the documentation",
        "completed": true,
        "assignedTo": "Unassigned",
        "project_id": null,
        "createdAt": "2025-11-07T12:35:13.123Z",
        "updatedAt": "2025-11-07T12:40:00.456Z"
    }
]
```

#### Example cURL

```bash
curl https://your-api-url.onrender.com/tasks
```

### 3. Get a Single Task by ID

Retrieves a single task by its unique ID.

-   **HTTP Method:** `GET`
-   **URL:** `/tasks/:id`
-   **URL Parameters:** `id` (The `_id` of the task)

#### Success Response

-   **Code:** `200 OK`
-   **Content:** A single task object.

```json
{
    "_id": "64a2f8b8c5d1e2f3a4b5c6d7",
    "description": "Design the new application logo",
    "completed": false,
    "assignedTo": "Alice",
    "project_id": "636d8b53a89366a315355607",
    "createdAt": "2025-11-07T12:34:56.789Z",
    "updatedAt": "2025-11-07T12:34:56.789Z"
}
```

#### Error Response

-   **Code:** `404 Not Found` if a task with the given ID does not exist.

#### Example cURL

```bash
curl https://your-api-url.onrender.com/tasks/64a2f8b8c5d1e2f3a4b5c6d7
```

---

### 4. Update a Task

Updates one or more fields of an existing task.

-   **HTTP Method:** `PATCH`
-   **URL:** `/tasks/:id`
-   **URL Parameters:** `id` (The `_id` of the task to update)
-   **Request Body:** A JSON object with the fields to update.

#### Request Body Example

```json
{
  "completed": true,
  "assignedTo": "Bob"
}
```

#### Success Response

-   **Code:** `200 OK`
-   **Content:** The full, updated task object.

```json
{
    "_id": "64a2f8b8c5d1e2f3a4b5c6d7",
    "description": "Design the new application logo",
    "completed": true,
    "assignedTo": "Bob",
    "project_id": "636d8b53a89366a315355607",
    "createdAt": "2025-11-07T12:34:56.789Z",
    "updatedAt": "2025-11-07T12:55:01.999Z"
}
```

#### Error Response

-   **Code:** `404 Not Found` if a task with the given ID does not exist.

#### Example cURL

```bash
curl -X PATCH -H "Content-Type: application/json" \
-d '{"completed":true}' \
https://your-api-url.onrender.com/tasks/64a2f8b8c5d1e2f3a4b5c6d7
```

---

### 5. Delete a Task

Deletes a task by its unique ID.

-   **HTTP Method:** `DELETE`
-   **URL:** `/tasks/:id`
-   **URL Parameters:** `id` (The `_id` of the task to delete)

#### Success Response

-   **Code:** `200 OK`
-   **Content:** A confirmation message.

```json
{
  "message": "Task deleted successfully"
}
```

#### Error Response

-   **Code:** `404 Not Found` if a task with the given ID does not exist.

#### Example cURL

```bash
curl -X DELETE https://your-api-url.onrender.com/tasks/64a2f8b8c5d1e2f3a4b5c6d7
```
